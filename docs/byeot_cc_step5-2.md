# 클코 작업 요청 — ⑤-2 식물 위치 판정 + 적합맵 실시간갱신

> tool.html 식물 진단을 "방 전체" → 놓은 자리 기준으로.
> + 적합맵 실시간 갱신 버그 수정.
> ★ "식물을 자리에 놓고 그 자리 빛으로 판정"이 게임 첫 루프의 씨앗 →
>   게임 이전에 미리 구조 잡기(식물 = 위치를 가진 객체).

---

## 현재 상태 (4501aa5 기준)
```
✅ 잘 됨: lux→PPFD(lumIllumAt), 광원종류 축, 3층 판정, 적합맵 색(fitColor)
⚠ 문제1: 적합맵이 실시간 갱신 안 됨 (토글 껐다 켜야 반영)
⚠ 문제2: 판정이 "방 전체 최대/최소 PPFD" 기준 → 식물 위치 개념 없음
```

---

## STEP 1 — 적합맵 실시간 갱신 (버그 수정)
```
증상: 광속·천공조도·식물선택·조명 바꿔도 바닥 적합맵이 즉시 안 바뀜.
원인: compute()가 drawHeat() 부르지만 S.showFit 상태 갱신 타이밍 어긋남.
고칠 것:
- compute() 끝에서 S.showFit이면 drawHeat() 반영 확실히(needsUpdate).
- plantSel·litHours 변경 때도 (적합맵 켜져 있으면) drawHeat() 재호출.
- "광원/천공/식물/조명 뭐든 바뀌면 적합맵도 즉시".
```

---

## STEP 2 — ★ 식물 위치 판정 (핵심, 게임 이전 전제)

### 개념
```
지금: "방 어딘가 밝으면 적합" (방 전체 max/min)
→   : 식물을 특정 자리에 놓고, 그 자리 PPFD로 판정
     "이 자리, 몬스테라 딱 좋아요 ⭐"
```

### 데이터 구조 (게임 재사용)
```javascript
S.plants = [];   // { id, x, z, y(높이=바닥0 or 선반위) }
// 지금은 1개만 놓아도 되지만 배열 → 게임에서 여러 식물 배치로 확장
```

### 배치 UX
```
- 식물 선택 → "여기 놓기" → 바닥 클릭
- 클릭 지점에 식물 마커(화분+잎)
- 놓인 위치 PPFD 계산 → 그 자리 기준 판정
- 마커 드래그 이동(가구 드래그 로직 재사용)
- 선반 위 클릭 → 그 높이 기준(높이별 조도 반영), 식물 y=선반 top
```

### 판정 (놓은 자리 기준)
```
- 놓은 지점 (x,z,y)의 PPFD = ppfdAt(x,z, y)
- needPPFD와 비교 → A/B/C층 판정 그대로
- 문구: "이 자리, {식물} 딱 좋아요 ⭐ / 적합 ✅ / 부족 🥶 / 과광 ⚠"
- 정밀: 그 자리 PPFD / 적정 / DLI
```

### 적합맵과의 관계
```
- 적합맵(바닥 초록/파랑/빨강) = 유지(좋은 자리 가이드)
- 마커 놓으면 그 지점 판정 우선
- 마커 없으면 "식물을 놓아보세요" 안내(또는 방 전체 폴백)
→ "적합맵 보고 → 거기 놓고 → 그 자리 판정" = 육성 루프 씨앗
```

## STEP 3 — 게임 이전 힌트 (구조만)
```
- S.plants[] = 게임 "심은 식물" 목록으로 그대로
- ppfdAt(위치)→needPPFD = 게임 "이 식물 이 자리 행복?" 판정
- 지금은 "적합/부족"까지만. 게임에선 성장속도·행복도로 (구현 X, 구조만)
```

## 핵심 원칙
```
- pointIllum · lux→PPFD(lumIllumAt) · A/B/C 판정 건드리지 마
- 데이터 읽기 전용
- 식물 마커는 가구 드래그/픽킹 재사용
- 적합맵 계속 실시간
- 각 단계 브라우저 확인 후 커밋·푸시
```
