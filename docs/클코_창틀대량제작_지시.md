# 클코 지시 — 창틀 프리셋 라인업 대량 제작 + 검수

> 전제: `buildWindowFrame` 확장(pattern·재질·유리 분기)과 `data/window_presets.json` 구조가
> 이미 구현·검증됐다. (안 됐으면 `클코_창틀프리셋_지시.md` 먼저.)
> 이 창은 에셋/방 전담. 로직 파일 안 건드림. 커밋은 내가 시킬 때만, `git pull` 먼저.

---

## 목표
창틀은 코드 모듈이라 GLB처럼 개당 비용이 안 든다 → **파라미터 조합으로 라인업을 대량 정의**하고,
한 번에 **컨택시트(썸네일 그리드)로 쭉 렌더해서 눈으로 검수**한다.
"한 방씩 실행" 대신 "전체를 한 판에 깔아놓고 고르기"가 코드 에셋의 장점.

---

## STEP 1. 파라미터 조합으로 라인업 확장
카탈로그 7개(win_studio_cross 등)를 기준으로, 아래 축을 조합해 **후보 프리셋을 늘린다**.
막 뽑지 말고 **의미 있는 조합만** (방에 실제로 쓸 법한 것):

### 조합 축
- **pattern**: grid / cross / letterbox / slim / curtainwall / none  (6)
- **frameColor**: 크림 `#f8f4ec` / 화이트 `#ffffff` / 우드 `#c9a36a` / 블랙 `#3a3a3a` / 민트 `#bfe3d0` / 연회색 `#d9dde0`  (6)
- **glass.type**: clear / green-tint / frosted / glassblock / none  (5)
- **gloss**: matte / satin / gloss  (3)
- **corner**: sharp / round  (2)
- **sill**: on / off  (2)

전 조합(6×6×5×3×2×2 = 2160)은 과함. **큐레이션 규칙**으로 추린다:

### 큐레이션 규칙 (무의미 조합 배제)
- 방 종류별로 **"어울리는 창"만** 만든다. 아래 STEP 2 방 매핑 기준.
- 색은 방당 1~2개 팔레트만 (파스텔 톤 유지, 블랙은 모던/카페 한정).
- glassblock·frosted는 조도 낮은 방(반지하·욕실st)에 몰아줌.
- round corner는 카페·아기자기 방 한정. 일반 주거는 sharp.
- 결과 목표: **대략 12~18개 프리셋** (2000개 아님). 각각 "이 방의 이 창" 하고 이름이 붙는 것.

---

## STEP 2. 방 라인업 ↔ 창 매핑 (이 게임 세계관 기준)
방(이사 진행: 반지하→원룸→아파트→단독/온실 등)마다 창을 배정:

| 방 | 창 프리셋(예) | pattern | 유리 | 빛 |
|---|---|---|---|---|
| 반지하 | win_semi_letterbox | letterbox 3×1 | frosted | 낮음(확산) |
| 반지하(레트로) | win_block_retro | grid 3×3 | glassblock | 낮음(산란) |
| 원룸 | win_studio_cross | cross 2×2 | clear | 중상 |
| 원룸(우드) | win_studio_wood | cross 2×2, 우드 | clear | 중상 |
| 아파트 거실 | win_apt_big | grid 3×2, sill | clear | 높음 |
| 학원/사무 | win_academy_slim | slim 1×2 | green-tint | 중 |
| 카페/모던 | win_cafe_clean | none 통유리, round | clear | 높음 |
| 온실/베란다 | win_greenhouse_wall | curtainwall 3×2 | clear | 최상 |
| 베란다(개방) | win_veranda_open | none | none(뚫림) | 최대(직사) |

> 방 라인업이 아직 안 정해진 건 **비워둬**. 확정된 방만 창 만든다(한 번에 하나씩).
> 지금 확실한 건 반지하·원룸 → 이 둘은 반드시. 나머지는 초안으로만.

## STEP 3. window_presets.json에 전부 기록
- 각 프리셋 = id + 전체 파라미터(STEP 1 축 값) + `glass` 블록 + `note`(어느 방/컨셉).
- `light` 힌트도 같이: `{ illumMul, diffuse }` (조도 계산이 나중에 읽음, 값만 보존).
- 크기(w·h)는 프리셋에 넣지 않음 — 방 쪽에서 지정(프리셋=룩/유리, 방=크기/위치).

## STEP 4. ★ 컨택시트 렌더 (핵심 산출물)
`assets/house/_window_contact.html` 생성:
- 모든 프리셋을 **작은 미니 씬으로 나란히 렌더** (썸네일 그리드).
- 각 셀: 창틀 하나 + 라벨(id·pattern·유리·색) + 빛 힌트.
- 배경 통일(연한 파스텔 벽 한 조각 앞에 창 세워서), 카메라 각도 통일.
- WebGL 실렌더 이 환경 불가하니 → **html+js로 만들어두고, 내가 로컬에서 열어 확인**
  (`python -m http.server 8777` → `_window_contact.html`).
- 이거 하나 열면 전 창틀을 한눈에 비교 가능하게. needs_review는 테두리 강조.

## STEP 5. 검수 루프 (내가 컨택시트 보고 낙점)
- 컨택시트 넘겨주면 내가 보고 "이건 살, 저건 두께 줄여, 색 바꿔" 피드백.
- 피드백은 **숫자 조정**(FT/BT/색/roughness)이라 빠름. GLB 재생성 아님.
- OK된 것만 `final:true` 마킹.

---

## ★ 원칙 재확인 (안 지키면 또 산으로 감)
- **2000개 조합 다 만들지 마.** 큐레이션해서 12~18개. 방에 붙는 것만.
- 방 라인업 확정 안 된 창은 초안(note에 "미확정")으로만, 렌더 검수는 반지하·원룸 먼저.
- 재질은 매끈 무광 파스텔 기준(어제 "투박·스티로폼" 문제 반복 금지).
- 조도 계산 실연결은 로직 창 일 — 여기선 light 힌트 값만 저장.
