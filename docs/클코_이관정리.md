## 1. 확정 사항 (더 안 흔들 것)

| 항목 | 결정 |
|------|------|
| 정체성 | 도구+게임 둘 다 목표. 재미로, 수익은 덤 |
| 개발 순서 | **엔진(빛·건축·조도) 먼저**, 게임은 항상 염두 |
| 기술 스택 | **웹 (HTML/CSS/JavaScript)** |
| 배포 | 웹 → 필요시 Capacitor(앱)/Electron(exe)로 포장 |
| 저장소 | GitHub 모노레포 1개 + collaborator 1명 |
| 문서/데이터 | Google Drive (스프레드시트로 식물·룩스 데이터) |
| 커넥터 | GitHub 먼저 익히고 → Google Drive 추가 (하나씩) |
| 오프라인/온라인 | 싱글=오프라인 / 거래·랭킹=온라인(서버, 나중) |
| 치트 방어 | 스택 아님. "중요 계산은 서버에서" (소셜 넣을 때) |

## 3. 첫 작업 순서 (클코에서 할 일)

```
① 레포 초기화
   - README.md (프로젝트 소개·구조)
   - .gitignore (웹/Node 템플릿)
   - 폴더 구조 잡기 (아래 4번)

② 프로토타입 쪼개기 (단계 A)
   - 볕_v2 한 파일 → 역할별 파일로 분리
   - lighting.js(조도엔진) / room.js(렌더) / plants.js / data(카탈로그)

③ 엔진부터 정돈 (개발순서 = 엔진 먼저)
   - 조도 계산 모듈을 깨끗이 분리·정리
   - 데이터(식물·조명·창문) → 별도 파일 or Google Sheet 연동

④ Git 기본흐름 연습
   - 브랜치 나누기 / 커밋 / 풀 리퀘스트
   - 협업자와 분담 규칙 정하기
```

```
byeot/                        (레포 루트)

│  │  └─ furniture.js         가구·배치
│  │
│  ├─ plants/
│  │  ├─ plant.js             식물 성장 로직
│  │  └─ genetics.js          (나중) 무늬 유전·번식
│  │
│  ├─ ui/
│  │  ├─ controls.js          슬라이더·버튼·회전

│  │
│  └─ main.js                 전체 조립
│
├─ data/                      ★ 데이터화 (dB)
│  ├─ catalog.json            아이템 카탈로그 (식물·조명·가구)
```

## 5. README.md 에 들어갈 것 (초안 뼈대)

```
# 볕 (가제)

빛이 진짜인 방꾸미기·식물육성 — 조도 시뮬레이터 + 식집사 게임

## 무엇
- 본체: 조명·창문 채광·룩스(조도) 시뮬레이션
- 부가: 방 꾸미기 + 식물 육성 (빛 따라 자람)

## 스택
- 웹 (HTML/CSS/JavaScript)
- (배포) Capacitor→앱 / Electron→exe (예정)

## 구조
- src/engine : 조도 계산 엔진 (먼저 개발)
- src/room   : 방 렌더링·8방회전
- src/plants : 식물 로직
- data       : 식물·아이템 데이터

## 개발 순서
1. 엔진(빛·조도) 먼저
2. 그 위에 게임(육성·꾸미기)

## 협업
- 브랜치 나눠 작업 → PR → 병합
```

## 8. 잊지 말 원칙 (흔들리면 기획트리 헌법 참고)

- 재미로 하는 것. 수익은 덤.
- 엔진 먼저, 게임 항상 염두.
- 얕게는 누구나 / 깊게는 덕후만.
- 한 번에 하나씩 (재미 확인 후 다음).
- 데이터는 깊게, 기본플레이는 단순 (엔지니어 함정 경계).
- 스택 안 바꿈 = 처음부터 웹 제대로.
