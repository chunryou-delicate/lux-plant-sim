# 게임화 착수 — 에셋 먼저 (2026-07-12 방향 전환)

> ★ 중대한 방향 전환: "우리는 시뮬레이터가 아니라 게임을 만든다"
> 지금까지 = 엔진(조도계산·성장로직) 100%, 에셋 0% → 시뮬레이터처럼 느껴짐
> 이 게임 종류 = 에셋(보이는 것) 70% + 로직 30%
> → 에셋(캐릭터·귀여움·이쁨성) 먼저 확보하고, 그 다음 로직·스토리로.

---

## 왜 방향 전환 (반발함의 정체)
```
"하루 넘기기 + 성장 숫자" = 로직은 도는데 감흥이 없음 = 시뮬레이터
게임이 빠지는 이유 = 정확함 X → 캐릭터·귀여움·감정이입 O
  (동물의숲=귀여운 주민+내 공간 / 다마고치=픽셀인데 죽으면 슬픔)
→ 엔진은 이미 튼튼. 이제 "게임"의 그 위에 살려야.
```

### 레퍼런스 (뽑아낼 지목) — 각각 뭘 가져올까
```
갓냥이하우스        → 캐릭터 귀여움 + 소소한 연출 ("캐릭터가 살아있다")
레인워             → 식물 키움 순환·판매 (우리 식물+경제)
동물의숲            → 내 공간 꾸미기 + 정든 주민 + 느긋한 힐링
류류 고양이리조트   → 귀여운 걸 모으고 배치, 늘어나는 재미 (수집·방치)
```

### 그래픽 목표
```
- room_3d_enclosed 수준이면 충분 (이 게임들도 저폴리 아기자기)
- 직교 4방위 카메라(room_3d_ortho.html) = 그 아기자기 룩
- 빛이 진짜라 오히려 더 예쁠 수 있음 (창가 빛 떨어짐)
```

## 에셋 확보 = AI 파이프라인 (미술가 아니어도 OK)

> 검증됨(2026 표준): AI로 원화(2D) 뽑고 → AI로 3D 변환. 그림 실력·툴 불필요.

### 파이프라인
```
1. AI로 원화(2D) 뽑기
   - 미드저니/DALL-E/Scenario 등, 아이소 스타일, 톤 통일
   - 배경 깔끔·주제 선명하게 (흐리면 3D 지저분해짐)
   - → 캐릭터·주요 오브젝트 = 4각도(앞뒤좌우) 뽑기 (멀티뷰용)

2. Tripo로 3D 변환 (추천 툴)
   - 단일 이미지 → 저폴리 GLB (Smart Low Poly, 모바일 5K face 이하)
   - 캐릭터 = 멀티뷰(4각도) → 일관된 3D + 자동 리깅(Rigging v2.5)
     · 사람 캐릭터는 이걸로 해결 (제일 어려운 부분)
   - GLB export

3. Three.js 로드
   - 직교 4방위 카메라로 예쁜 각도만 노출
   - AI 3D 약점(뒷면·바닥 뭉개짐)은 카메라가 가려줌 = 우리 유리점
```

### 주의 (검색 확인)
```
- 입력 이미지 깨끗해야 좋은 3D
- 모바일용 저폴리로 (Smart Low Poly, face 제한)
- 토폴로지·UV 지저분할 수 있음 → 확인 필요
- 상업 출시 = Tripo 유료 플랜(생성물 상업 소유권). 무료로 먼저 테스트
- 스타일 통일 중요해지면 Scenario(내 아트 학습)도 옵션
- 크레딧 관리 (다 뽑지 말고 MVP 세트만)
```

## MVP 에셋 세트 (이만큼만 뽑고 멈춤) ★

> 다 만들지 말 것. 최소 세트로 "게임처럼 보이는" 확인 후 로직·스토리로.

```
캐릭터      1~2개   (예: 자취자 1 + 1)
식물        4~5개   (몬스테라·포토스·산세베리아·다육·바질 등 대표종)
집          1~2개   (예: 반지하 1 + 원룸 1, 빛 등급 다르게)
인테리어    1~2개   (벽지·바닥 세트)
가구        4~5개   (침대·책상·선반·의자·러그 등)
아이템      4~5개   (화분·물뿌리개·비료·장식 등)
GUI         1세트   (버튼·패널·아이콘·폰트 = 게임 인상 좌우)
```

### 각 에셋 성장/상태 고려
```
- 식물: 성장단계별(씨앗·새싹·성체) → 같은 스타일로 3~4단계
  · 프롬프트 변형(seedling/grown)으로 뽑되 스타일 통일
- 캐릭터: 기본 포즈 + (가능하면) 간단 동작
```

## 순서 (이번 단계)
```
1. 스타일 확정    → 원화 몇 장 뽑아 "이 톤이다" 정하기 (제일 중요)
2. MVP 에셋 뽑기 → 위 세트만. 캐릭터·식물부터.
3. GUI 뽑기       → 게임 화면 껍데기
4. Three.js 통합 → 에셋 로드, 카메라, 배치
──── 여기서 멈춤 ────
5. 로직·스토리로 복귀
   - 에셋 위에 얹으니 로직이 "살아있게" 붙음
   - 성장로직(이미 있음) + 경제 + 캐릭터 서사 + 이사
```
→ "게임처럼 보이는" 확인이 이 단계 목표.
  에셋 붙였는데도 반발하면 → 살아있는 연출(흔들·반짝·회전) 추가.
