# 볕 집(Room) 에셋 — 코드 창 인수인계서 (2026-07-19 · 에셋 단계 완료)

> **이 문서 하나만 보면 main.js가 집 모듈을 구현할 수 있게 정리함.**
> 에셋(창호·문·텍스처) 제작은 여기서 종료. 남은 건 전부 코드 작업(§7).

---

## 1. 스타일 = **A 미니멀** (확정)
- 순수 카툰, 매끈·중립. 무텍스처에 가까운 솔리드 파스텔.
- **온기·분위기는 에셋에 굽지 않는다.** 태양·조명 = **게임 빛 시스템(런타임)** 으로 실시간 표현.
- (B_웜·C_동숲은 따뜻한 빛이 이미 원화에 구워져 런타임 조명과 충돌 → 배제.)

## 2. 집 = **모듈 방식** (확정)
- **뼈대(벽·바닥·천장) = 코드 생성**(main.js). 에셋 아님. 박스 지오메트리 + 텍스처.
- **에셋 = 창호·문(3D GLB) + 벽지·바닥재(2D seamless 텍스처).**
- 집 5종의 차이 = **모듈 조합 + 텍스처 + 창 크기·위치**로 표현.

## 3. ★ 유리 = 프레임만 3D + 런타임 셰이더 (핵심)
**검증됨: Meshy는 유리를 항상 불투명으로 구움(alphaMode=OPAQUE). 두 번 확인(병·창호).**
→ 그래서 창호 GLB는 **유리 없는 프레임만**으로 제작. 유리 자리는 **뻥 뚫려 있음**.

**코드가 할 일**: 프레임 GLB를 로드하고, 개구부(뚫린 사각)에 유리 quad(PlaneGeometry)를 코드로 끼운 뒤 투명 재질 부여:
```js
const glass = new THREE.MeshPhysicalMaterial({
  transmission: 1.0, roughness: 0.0, thickness: 0.5,
  transparent: true, ior: 1.5, opacity: 0.15,
  color: 0xcfe8ff        // 살짝 블루틴트(선택)
});
```
- 프레임 = GLB 그대로(크림/화이트), 유리 quad = 위 재질로 별도 생성해 개구부 위치에 배치.
- 개구부 크기·위치는 GLB 프레임 안쪽 치수에서 계산하거나 코드 상수로 정의.

## 7. 남은 작업 = **전부 코드 창(main.js)**
1. **벽·바닥·천장 박스 지오메트리** 코드 생성 + 텍스처(§5) RepeatWrapping 입히기.
2. **유리 셰이더**: 창 프레임 GLB 개구부에 투명 quad 끼우기(§3).
3. **창/문 GLB 배치**: 벽 구멍에 프레임 끼움(위치·스케일 상수화).
4. **샘플 방 조립**: 반지하 1채 먼저(창-반지하+시멘트 텍스처+낮은 조도)로 파이프라인 검증.
5. **빛 공학 연동**: 창 크기·위치·방향을 조도 계산 입력으로.
6. **후행 부품(몰딩·걸레받이 등)**: 필요 시 별도 3D — 에셋 창에 재요청.
