# 클코 지시 — 창틀 고정 에셋(프리셋) 시스템 구현

> 이 창은 **에셋/방 전담**. 로직 파일(plant_grow.html·tool.html 등) 안 건드림.
> 커밋은 내가 "커밋해" 할 때만. 자동 커밋 금지, 커밋 전 `git pull` 먼저.

---

## 배경 (이미 된 것)
- 창틀은 이제 GLB가 아니라 **코드 모듈** `buildWindowFrame(w, h, cols, rows, mat)`이야. 반듯하게 잘 나옴.
- 방·벽·바닥·천장·문도 다 코드. 유리는 셰이더 투명.
- 지금 창틀이 파라미터로 굴러가니까, **"고정 창틀 에셋"을 프리셋 데이터로 정의**하고 싶어.
  각 프리셋 = 룩(두께·형태·색·유리·광택·패턴) + 빛 특성(조도 영향) 한 묶음.

---

## 할 일 (순서대로)

### STEP 1. `buildWindowFrame` 파라미터 확장
현재 `(w, h, cols, rows, mat)`에 아래 옵션 추가. 하나의 옵션 객체로 받게 리팩터해도 좋음:

```
buildWindowFrame(w, h, opts)
  opts = {
    cols, rows,              // 격자 분할 (기존)
    FT=0.09,                 // 바깥틀 두께
    BT=0.045,                // 격자살 두께
    depth=0.14,              // 프레임 깊이
    corner='sharp',          // 'sharp' | 'round'  (round=모서리 살짝 둥글게, 동숲st)
    pattern='grid',          // ★ 아래 STEP 2 참조
    sill=false,              // 창턱(선반) 유무 — 식물 올려두는 자리
    frameColor='#f8f4ec',    // 프레임 색
    gloss='matte',           // 'matte' | 'satin' | 'gloss'
    glass={...}              // STEP 3 참조
  }
```
- `sill:true`면 창 하단에 얇은 선반(박스) 추가. 폭은 창 너비, 깊이 얇게(0.12쯤), 벽 안쪽으로 튀어나오게.
- `gloss`는 재질 roughness/metalness로 매핑 (상세 수치는 STEP 3-B 재질 섹션).
- `corner:'round'`는 프레임 바 모서리를 살짝 베벨/라운드. 과하지 않게.

### STEP 2. ★ 패턴(격자 배치) — 이게 핵심 추가
`pattern`으로 격자 살 배치 방식을 바꾼다. cols/rows만으로 안 되는 배치까지 커버:

| pattern | 배치 | 비고 |
|---|---|---|
| `grid` | cols×rows 균등 격자 | 기본. 살이 가로·세로 균등 |
| `cross` | 십자 1개 (가운데 수직살+수평살) | 2×2처럼 보이되 정중앙 십자 |
| `letterbox` | 세로살만 (가로 긴 3분할 등) | 반지하 가로창. rows 무시, cols-1개 세로살 |
| `slim` | 가로살만 (세로 좁은 창) | cols 무시, rows-1개 가로살 |
| `curtainwall` | 멀리언 굵게, 3×2 유리벽 | 온실 통창. 살 좀 더 두껍게 |
| `none` | 살 없음, 통유리 | 바깥틀만, 격자 없음 |

- 각 pattern이 격자살 지오메트리를 어떻게 배치할지 함수 안에서 분기.
- `letterbox`/`slim`은 한 방향 살만 생성. `none`은 바깥틀만 만들고 격자 루프 스킵.
- `curtainwall`은 BT를 자체적으로 좀 키워(예: ×1.4) 멀리언 느낌.

### STEP 3. 유리 프리셋 (빛과 직결)
유리 quad 재질을 `glass.type`별로 분기:

```
glass = {
  type: 'clear',        // 'clear' | 'green-tint' | 'frosted' | 'glassblock' | 'none'
  transmittance: 0.92,  // 0~1, 조도 계수용
  tintColor: null,      // '#a8d8c0' 같은 색조 or null
  glossy: 0.3           // 유리 반사/광택 (룩용)
}
```
- `clear` = 맑고 투명, 살짝 반사.
- `green-tint`/tintColor = 색조 살짝 입힘.
- `frosted` = 반투명 확산(불투명도 올리고 blur 느낌 — 뒤가 흐릿).
- `glassblock` = 격자진 반투명 블럭 패턴 + 강한 산란.
- `none` = 유리 quad 아예 안 만듦(뻥 뚫린 창/베란다).
- ★ **transmittance는 나중에 조도 계산이 읽을 값이니 재질에 저장(userData)해둬.** 지금 빛 계산에 연결까진 안 해도 됨. 값만 보존.

### STEP 3-B. ★ 재질(material) 상세 — 프레임·유리 실제 파라미터
룩이 원화(A 미니멀 파스텔)랑 맞으려면 재질값이 중요해. Three.js 재질을 아래 기준으로 만들어.
파스텔 저폴리·동숲st = **매끈·무광 위주, 반사 과하지 않게, 결(그레인) 없음.**

**(1) 프레임 재질 — `MeshStandardMaterial`**
`gloss` 값 → roughness/metalness 매핑:

| gloss | roughness | metalness | 느낌 |
|---|---|---|---|
| `matte` | 0.9 | 0.0 | 무광 크림. 기본. 스티로폼 아니고 매끈한 무광 |
| `satin` | 0.55 | 0.0 | 살짝 부드러운 광 (도장면st) |
| `gloss` | 0.2 | 0.05 | 도자기st 광택 |

- `color` = frameColor 그대로. `flatShading:false`(부드러운 셰이딩, 저폴리라도 각지지 않게).
- **텍스처(맵) 안 씀** — 단색 재질. 결·노이즈 넣지 마 (원화가 매끈 단색이라). 나중에 옅은 결 필요하면 그때.
- envMapIntensity 낮게(0.3~0.5) — 반사 튀지 않게. 파스텔 톤 유지.
- 문틀·문·손잡이도 같은 프레임 재질 규칙 따름(손잡이만 gloss 살짝 올려도 됨).

**(2) 유리 재질 — type별로 다른 재질 생성**
가능하면 `MeshPhysicalMaterial`(transmission 지원). 안 되면 `MeshStandardMaterial`+opacity로 폴백.

| type | 재질 핵심값 | 룩 |
|---|---|---|
| `clear` | transmission 0.9 / roughness 0.05 / opacity 0.25 / ior 1.5 | 맑고 투명, 살짝 반사 |
| `green-tint` | 위 + `color=tintColor`(#c8ded0) / transmission 0.8 | 옅은 초록 유리 |
| `frosted` | roughness 0.9 / transmission 0.5 / opacity 0.6 / 뒤 흐릿 | 간유리(확산·불투명) |
| `glassblock` | roughness 0.7 / opacity 0.7 / 블럭 격자 노멀or살짝 요철 | 유리블럭 산란 |
| `none` | 재질·quad 생성 안 함 | 뻥 뚫림 |

공통:
- `transparent:true`, `depthWrite:false` (투명 정렬 깨짐 방지).
- `side: THREE.DoubleSide` (안팎 다 보이게).
- MeshPhysicalMaterial 미지원 환경 대비: transmission 없으면 `opacity`만으로 반투명 흉내 + roughness로 frosted 구분.
- **frosted/glassblock 흐릿함**: transmission이 안 먹으면 opacity 올리고 뒤 살짝 안 비치게. blur까진 셰이더 필요하니 opacity로만 근사.
- `userData.transmittance` = glass.transmittance 저장 (조도 계산용, 값만).
- `userData.diffuse` = (frosted·glassblock이면 true) 저장 — 나중에 확산광 처리 플래그.

**(3) 재질 재사용 (성능)**
- 같은 프리셋/색/gloss 조합은 재질 인스턴스 **캐싱**해서 공유(매 창마다 new 하지 말 것).
- `matCache[key]` 식으로. 방에 창 여러 개여도 재질은 몇 개만.

### STEP 5. house_rooms.json 연결
- 각 방 창 정의에서 `pattern`/`cols`/`rows`를 직접 쓰던 걸, **`preset: "win_studio_cross"` 처럼 프리셋 id 참조**로 바꿔도 되게.
- 프리셋 id가 있으면 프리셋값 로드, 없으면 기존처럼 인라인 값 사용(하위호환).
- w·h(창 크기)는 방마다 다르니 프리셋엔 안 넣음 — 방 쪽에서 지정. (프리셋=룩/유리, 방=크기/위치)

## 산출·확인
1. `buildWindowFrame` 확장 + pattern 분기 + 유리 프리셋 분기 구현.
2. `data/window_presets.json` 생성.
3. house_rooms.json에서 원룸·반지하 창을 프리셋 참조로 전환.
4. WebGL 실렌더는 이 환경 불가하니 문법·JSON 검증까지 하고, 나한테 로컬 확인 방법 알려줘
   (`python -m http.server 8777`). 원룸·반지하 창이 어떻게 나오는지 볼게.
5. 확인할 것 짚어줘:
   - pattern별(cross/letterbox) 살 배치 맞나
   - **프레임 재질** — matte가 무광 크림으로 깔끔한가(스티로폼처럼 밋밋하지도, 반사 튀지도 않게), 색이 파스텔인가
   - **유리 재질** — clear가 맑게 비치나, frosted가 흐릿하게 확산되나(뒤 흐려짐), green-tint 색조 보이나
   - sill 없이 깔끔한가, 두께(FT/BT) 원화 대비 어떤가
   - MeshPhysicalMaterial transmission이 이 Three.js 버전에서 먹는지(안 먹으면 opacity 폴백 됐는지)
