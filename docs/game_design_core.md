# 게임 설계 핵심 — 목적·경제·캐릭터·이사 (2026-07-12)

> "빛이 진짜"라는 엔진 위에 얹을 게임의 뼈대.
> 왜 계속 하는가(목적) + 어떻게 굴러가는가(경제) + 누구로 하는가(캐릭터) + 이동성(이사).
> ⚠ 설계는 다 이어두되, 구현은 한 번에 하나씩(재미 확인 후 다음).

---

## 1. 게임의 두 축 (모든 게임 뼈대)
```
① 생존 축 (벌)       → 못 벌면 파산. 단기 동기.
② 성취발현 축 (성취) → 목표 달성하면 승리. 장기 동기.

→ 캐릭터마다 두 축의 "비중"이 다름 = 그게 곧 난이도이자 정체성.
→ 돈은 단기 긴장, 성취발현은 장기 목표. 두 층 구조.
```

---

## 2. 캐릭터 = 다이얼 시스템 (핵심 구조)

> 캐릭터 선택 = 난이도 + 동기 + 승리조건 + 이사무게를 한 번에 선택.
> 시스템은 하나, 캐릭터는 "다이얼 설정값"만 다름 → 데이터로 확장.

### 캐릭터 4종 (초기)
```
캐릭터    수입원               경제압박  파산  메인목표          난이도
────────────────────────────────────────────────────────────────
주부      월급(생활비 커버)     없음     없음  누림가든·꾸밈      쉬움
연구자    고정월급(생활비커버)   없음     없음  희귀종·신품종      쉬움~보통
가장      식물부업+약간직업      중간     있음  목표금액→온실꿈    어려움
자취자    오직 식물팔이         높음     있음  생존→번식판매출    험
```

### 수입 구조가 곧 난이도 (별도 수치 불필요)
```
"다달이 돈 들어오냐 / 식물로만 버냐" = 압박 결정
→ 주부·연구자: 월급이 생활비 커버 → 지출 압박 없음 → 파산 없음
→ 가장: 부업이라 조율 필요 → 중간
→ 자취자: 전부 압력 → 험
```

### 각 캐릭터의 성취발현 = 승리 조건
```
주부   → 누림가든 완성 / 우리집 빛 마스터 / 사계절 컬렉션
연구자 → 도감 완성(희귀종 수집) / 신품종 창조(무늬·색) / 명성
가장   → 목표금액 달성 → 2단계 "나만의 온실 차리기"
자취자 → 좌절 모은 방 직화 탈출 → 온실 창업 (이사=승리)
※ 못 벌면 파산 실패 / 성취 못 이루면 "그런 삶" 실패
```

## 3. 공통 시스템 (모든 캐릭터 공유, 다이얼로 on/off·비중조절)
```
📖 도감 (수집·기록)   → 키운 식물 기록. 동물의숲 박물관 감성. ★공통 바닥
🏡 정원 완성도       → 공간이 얼마나 채워지고 예뻐졌는지 점수
🏆 명예/명성         → 희귀종·신품종·무늬 성공 누적 표현
💰 경제              → 수입(판매)·지출·월급
🎯 2단 목표          → 생존 넘기면 → 성취발현 개방 (돈→꿈 전환)
🌱 육성 루프         → 빛·물·환경 맞춰 성장 (엔진=조도 연결)
```

### 캐릭터별 다이얼 표 (뭘 메인으로)
```
             경제압박  도감    정원    명예    이사무게
주부          낮       O      O메인   약      쉬움
연구자        낮       O메인   약      O메인   쉬움(온실 지향)
가장          중       O      O       O       이사 핵심
자취자        험       O      약      O       ★탈출(이사=승리)
```

## 4. 이사 시스템 (성장 = 빛 환경 갱신)

> 이사 = 공간 확장 + 빛 환경 변화. 압박(파산)이 아니라 성장 보상.
> 캐릭터마다 무게만 다름.

### 집 등급 = 빛 환경 (후크와 직결) ★핵심
```
집               빛 환경               키울 수 있는 것
──────────────────────────────────────────────────
반지하           빛 최악(식물등 의존)   저광관엽만 / 자취자 시작
저층 원룸         한쪽 창, 제한적        저광~중광
2~3층           채광 보통             중광
남향 고층         빛 좋음, 직사광 O      밝은간접·직사
장미/테라스       강광                 다육·허브 천국
옥상/주택         최상(전부 가능)       모든 식물 / 성공 승리조건
──────────────────────────────────────────────────
→ 이사할 때마다 "이 집 빛은? 뭐 키우지?" 새로 고민 = 후크 갱신
→ tool(빛 진단)과 게임이 만나는 지점: 이사 온 집 빛 진단 → 식물 배치
```

### 이사의 무게 (캐릭터별)
```
주부·연구자 → 여유로운 업그레이드 ("돈 모이면 더 좋은 곳")
가장        → 꿈 목표(온실) 다가서는 이정표
자취자      → 생존 건 탈출 (반지하 → 상위 = 승리길)
※ 위 구조에서 자동으로 무게 결정 (별도 설정 불필요)
```

### 이사 트레이드오프 (경제 게임 심장)
```
좋은 집 = 빛↑ + 성취↑ BUT 비용↑ (자취자·가장 부담) — 단, 빛 좋으면 좋은식물·수익↑
→ "지금 이사? 더 모을까?" 고민 = 경제적 결정의 핵심
```

### 구현 순서 (방그리기 재사용 ★)
```
1단계 (지금): 집 프리셋 몇 개 + 프리셋 간 이사
   - 각 프리셋 = 빛 환경 다름 / 돈 모으면 더 밝은 집으로
2단계 (나중): 커스텀 이사 = tool 방그리기 재사용!
   - "이 집 직접 그려서 이사" = 도구 방그리기가 게임의 부품
   - 실제 우리집 넣기(트랙A)와 연결
   → "엔진 하나 껍데기 둘"이 이사에서도 실현
```

## 5. 데이터 구조 (한 줄로 확장)
```javascript
// 캐릭터 = 다이얼 설정값

  monthlyExpense: 50,        // 월세·생활비

  goalValue: 5000,           // 목표값

// 집 = 빛 환경 + 비용

// 식물 = 관상구 → data/plants.json (needPPFD 등)
```

## 6. 구현 순서 (엔지니어 함정 경계)
```
⚠ 다 넣을 수 있게 설계하되, 한 번에 하나씩 구현.
1. 캐릭터 1종 + 경제 1개로 재미 확인 (예: 자취자 + 판매·월세 = 긴장감?)
2. 재미있으면 → 도감 → 정원
3. 캐릭터 늘리기 (다이얼만 추가)
4. 이사(프리셋) → 커스텀 이사
5. 무늬·교배(Lv2) 등 깊이
→ "한 번에 하나씩, 재미 확인 후 다음" (헌법)
```
