## 1. DB 검증 결과 — 합격 ✅
```
품질: 우수. 지어낸 값 없음. 실측/추정/unknown 3층 정직 구분.
- 실측값이 상식과 일치 (몬스테라 100-250, 다육 400, 극락조 250-600…)
- 상충 정직 기록 (포토스·알로카시아 등 notes "확인필요")
- unknown 정직 비움 (알보·상추 = null + "리서치 필요")
- 광원: Apogee 공학계수(신뢰최상). 퍼플등 null+경고(정확).
- 통찰: 조명 = [설치형태]×[광원종류] 조합 (도구·게임 연결점)
```

### 데이터가 자연스럽게 3층 (→게임과 맞물림)
```
A층 실측 17종    → PPFD 정확     → "정밀 판정" (초보·흔한 것)
B층 fc만 30종    → level만 있음   → "대략 판정" (저광/중광/직사)
C층 unknown 32종 → 데이터 없음    → "미지의 식물" (희귀·고급)
```

## 2. unknown 32종 처리 = C의 승화 (공백을 게임으로)

> 데이터 공백을 "채우기"가 아니라 "게임 메커닉"으로 전환.
> 이유: 지어내면 신뢰 무너짐 + 마침 게임 깊이도 됨.
```
A층 (실측)   → 정밀 판정 "이 자리 몬스테라 딱 좋아요 (PPFD 180)"
B층 (fc만)   → 대략 판정 "이 자리 저광식물 OK, 정밀값 미확보"
C층 (unknown)→ 실험 메커닉 "데이터 없는 희귀종! 키우며 알아내세요"
              → 연구자 캐릭터의 핵심 재미 (미지 광량 발견)
              → 잘 키우면 "이 종의 적정 광 발견!" = 도감·명성 보상
```

### 게임 난이도와 자동 정렬
```
흔한 것(A) = 데이터 확실 = 초보 쉬움
희귀종(C)  = 데이터 없음 = 고급 도전 = 비싼게 팔림 (자취자·연구자)
→ "데이터 공백 = 희귀도 = 난이도 = 보상" 4박자 정렬
```
⚠ 단, 출시 초기 최소 라인업은 A층에서 (판정 확실해야 첫인상 좋음).
   C층은 "나중에 해금되는 도전 종"으로.

## 3. ⑤ 식물 적합성 판정 설계 (도구 종료선 마지막)

### 입력 → 판정 흐름
```
1. 자리 조도 계산 (tool 이미 있음: pointIllum → lux)
2. 광원 종류 확인 → lux_to_ppfd 계수 곱 → PPFD
   · 자연광(창): 0.0185
   · 조명: 광원별 계수 (light_sources.json)
   · ⚠ 퍼플등: lux 변환 금지 → 직접 PPFD 지정
3. (선택) PPFD × 시간 → DLI
4. 식물 needPPFD와 비교 → 판정
```

### 판정 로직 (식물 데이터층별)
```
A층 (ppfd 있음):
   PPFD < min      → "빛 부족 ❌"
   min ≤ PPFD ≤ max → "적합 ✅"
   opt 근처         → "최적 ⭐"
   PPFD > max      → "과광 ⚠"
B층 (level만):
   level 밴드로 대략 판정. 정밀 PPFD "미확보" 표시
C층 (unknown):
   "이 식물은 광요구 데이터가 없어요" → 실험 모드(키우며 관찰)
   → 성공 시 "적정 광 발견!" 기록
```

### UI 문구 (식집사용, 숫자 최소)
```
겉(기본): "이 자리, 몬스테라 딱 좋아요! ☀️" / "빛이 좀 부족해요 🥶"
속(디테일): PPFD 180 / 적정 100~250 / DLI 7.8 (상세설정 켤 때만)
→ "얕게 누구나 / 깊게 덕후만"
```
