# 볕 · 몬스테라 성장 순서도 — 클코 인계 문서

## 이 도구가 뭔가
`byeot_growth_chart.jsx` — 몬스테라 성장 **매커니즘을 시각적으로 정의**하는 순서도 에디터(React 단일 파일 아티팩트).
목적: 개발자(클코)와 기획자(박사님)가 "식물이 어떻게 자라는가"를 그림으로 합의하는 도구. 게임 런타임 코드가 아니라 **설계 명세서**임.
원본 레퍼런스: 박사님이 그린 성장 도감 PPT (밑동 아래 고정·생장점 위·왼→오 시간·위아래 분기).

## 핵심 개념
- **밑동(씨앗) = 화면 맨 아래 고정**, 생장점(파랑)은 위로 뻗음.
- **가로(→) = 시간 진행**: 성장 동작 하나 = 오른쪽에 새 스텝. 번호 #1, #2, #3…
- **세로(↓) = 가능성 분기**: 같은 시점의 다른 경우. 번호 #N-1, #N-2…
- 각 스텝은 그 시점의 식물 상태 스냅샷.

## 성장 규칙 (박사님이 실제 몬스테라 키우며 확정)
1. **초기 발아 순서**: 씨앗 → 얇은 줄기(잎자루) 나옴 → 그 끝에 말린 새순(주황) → 펴짐(노랑) → 어린잎(초록) → 중간잎(파랑). 줄기는 그 사이 계속 마디를 쌓으며 자람.
2. **줄기 상태 5종 (실제 3D 부품 매핑)**:
   - `petiole` 가는 잎자루(연두, 얇음)
   - `pink` 분홍대기 (혹이 날 가능성 품음. 이전 thin/med를 통합 — 50:50 랜덤이라 하나로)
   - `thick` 눈 생김/활성 (살색+빨간 점=눈. 여기서 가지가 남)
   - `main` 희귀(하늘색)
3. **눈(빨간 점)에서 프랙탈**: 눈 → 가는 잎자루 먼저 나옴(새순 없이) → 그 잎자루 끝에서 새순 → … 씨앗 발아와 **동일한 순서** 반복. 가지는 우상향 대각선.
4. **잎 성숙**: 새순→펴짐→어린잎→중간잎(mid)까지는 시간대로 자동. mid→성숙(갈라짐)은 확률.
5. **더블혹**: 순서도엔 **안 그림**(항상 1혹). 혹이 2개여도 각각 성장은 동일 매커니즘이라 분기 불필요 — 경우의 수 폭발 방지. 확률값(참고)으로만 종 확률 패널에 존재.
6. **★가능성 분기의 의미(중요)**: 줄기에 있는 **분홍 예비혹(pink) 각각이 "진짜 눈이 되는" 경우**가 하나의 분기. 예: 예비혹 5개 → 5개 분기. 몬스테라가 어느 마디에서 새 줄기를 낼지 랜덤인 규칙을 표현.

## 확률 (종 기본 + 지점 오버라이드)
`DEFAULT_PROB = { mid1:60, mature:20, special:8, matureSpecial:25, doubleBud:15 }`
- 확률 개입 지점: mid 단계(mid1/mid2 비율+특수발현) / 성숙 넘어가기(성숙+특수) / 더블혹.
- ★성숙 특수무늬는 앞선 mid의 특수 형태를 따라 그룹핑(일관성) — 미구현, 향후.
- 잎 클릭 → 지점별 확률 오버라이드 가능(비우면 종 기본값).

## 데이터 구조 (내보내기 JSON)
```
{
  version: 1, species: "monstera",
  tree: {
    root: "s0",
    nodes: {
      "<stepId>": {
        id, parent, children:[], col(시간), row(분기),
        plant: {
          root: "seed",
          nodes: {
            "<nodeId>": {
              id, up(위 마디 id), buds:[], stem, isTip(생장점), isSeed(밑동)
            }
          }
        }
      }
    }
  },
  prob: {...}
}
```
- **줄기 방향**: node.up = 위(생장점 방향). root(seed)가 밑동.
- **bud**: `{ stage(잎 0~4), side(±1), special, childNode(가지 root id), probOverride }`. childNode 있으면 그 bud는 가지(잎자루→…)로 뻗음.
- **stem 값**: petiole / pink / thick / main. (thin·med는 pink로 통합, 레거시 호환만)
- 번호(#N, #N-M)는 col/row로 렌더 시 계산(저장 안 함).

## 렌더 규칙 (layoutPlant)
- 밑동 y=0, up 갈수록 y 감소(위로). SEG=40.
- 가지(childNode)는 우상향 대각선(진행각 ± 0.6 rad).
- 눈(eye) 마디의 가지는 **빨간 점 위치**(몸통 중간 = 진행방향 반대로 SEG/2)에서 시작. `pos[id+"_eye"]`에 좌표 저장.
- 줄기 선(n→n.up) = 위 마디(n.up)의 몸통. 각 마디는 몸통 1개.
- 중간 마디 흰 원 = 클릭 비활성(표시만). 클릭 가능: 씨앗/생장점/줄기선/빨간점(눈)/잎.

## 조작 → 데이터 변화
- 발아: seed.isTip=false, 위에 petiole 마디(생장점) 추가.
- 줄기 자람: 생장점 아래에 petiole 마디 삽입(생장점이 위로).
- 새순: 생장점 bud 추가(항상 1개).
- 잎 성숙: bud.stage++.
- 줄기 상태: stem 변경. 눈 발현 시 thick.
- 눈→잎자루: 눈 마디 isTip 유지 안 함, bud에 childNode=새 petiole(생장점).
- 성장 동작 = 오른쪽 새 스텝(grow). 확률/특수 = 제자리(editHere). 분기 = 아래 새 갈래(branch).
- 마지막 스텝 삭제 = 현재(리프) 스텝 제거 후 부모로.

## 현재 상태 (byeot_with_branches.json, 50 스텝)
- #1~#10: 한 줄 성장(분기 없음).
- #11에서 첫 분기 → 2갈래(위/아래), 각각 다른 위치에 눈 1개.
- #11~#25: 2갈래 평행 성장.
- #26(col25): 각 갈래의 예비혹 5개 각각이 눈이 된 **10개 분기** 자동 생성됨(스크립트로).
  - 위 갈래 예비혹: 메인 3 + 가지 2 = 5개.
  - 각 분기 = 원래 눈 1 + 새 눈 1 = 눈 2개, 위치 다름.

## 향후 (미착수)
1. **스텝 간 텀(시간) 지정** — 각 스텝이 시간축 눈금. #N→#N+1 사이에 실제 소요 시간(며칠) 지정 → 성장 속도 데이터 → plant_grow 규칙. 빛이 이 속도를 조절.
2. 무늬 그룹핑(성숙 특수 일관성).
3. 색조정 파생 에셋 편집기(HSL로 잎 종류 다변화) — 별도.
