# 준비물 인벤토리 — 에셋 & DB (2026-07-12)

> 본격 구현 전, 미리 준비할 것(에셋)·쌓을 것(DB)을 한눈에.
> ⭐ 원칙: **다 모으고 시작 ❌ → 스키마(구조) 먼저 + 대표 씨앗 → 구현하며 채운다.**
> (엔지니어 함정 경계: 데이터는 깊게, 기본플레이는 단순. 새 아이템 = 데이터 한 줄.)

---

## ⚠ 먼저 — "싹 모으고 시작" vs "스키마 + 씨앗"
- 완전 수집 후 시작 = 끝없는 함정. 시작이 안 됨.
- 대신: 각 DB의 **컬럼(스키마)만 확정** → 대표 3~5개로 씨앗 → 굴리면서 확장.
- 에셋도: 필요 목록 확정 → 무료/기본 몇 개 → 늘림.

---

## A. 데이터베이스 (dB) — 깊이의 원천

### A1. 식물 DB ★가장 중요 (⑤ 적합성 직결)
```
스키마(안):
id, 이름, 카테고리(꽃/관엽/허브/다육/난/분재)
빛:   PPFD_min / PPFD_적정 / PPFD_max (µmol/m²/s)  또는 DLI 범위
      광질(직사/간접), 무늬유지_PPFD, 광포화·광저해
물:   주기, 과습위험   환경: 온도, 습도, 통풍
성장: 속도, 단계별비주얼, 무늬발현조건, 번식
가치: 희귀도, 판매가, 수확물
```
- 상태: ❌ (catalog.json에 needLux(lux)만 → **PPFD로 갈아타야**)
- 씨앗: 상추·바질 / 몬스테라·알보 / 다육 1종

### A2. 조명기구 DB ★스펙트럼 여기
```
종류(상부/매립/벽/기둥/가구/식물등), 광속(lm), 배광, 색온도(K),
스펙트럼: R:B비, 원적색 유무, lux→PPFD 계수(µmol/lux),
분류태그(백색LED/전구색/형광/할로겐/적청식물등/풀스펙트럼)
```
- 상태: 부분 (종류·배광·색온도·높이 O / **스펙트럼·PPFD계수 ❌**)

### A3. 창·채광 DB
- 방위(N/E/S/W), 크기, 유리 투과율, (심화) 지역별 기상(천공조도). 상태: 부분(천공조도 단일값 O)

### A4. 가구 DB (에셋과 연결)
- 크기·높이·**재질 반사율**·그림자박스·회전. 상태: 부분(치수·반사 개념 O / 재질별 반사율 ❌)

### A5. 방/집 프리셋 DB
- 원룸/투룸/L자, 창 위치, 천장고. 상태: 부분(room_presets.json 1개)

### A6. 스펙트럼 변환 테이블
- 광원별 lux→PPFD, 파장 밴드(청·녹·적·원적). 상태: ❌ (리서치 필요)

## B. 에셋 (시각 리소스) — 게임 무게중심
```
B1. 가구 저폴리 3D 모델 — 침대·책상·선반·의자·화분… (도구·게임 공용)
B2. 식물 모델 — 종별·성장단계별 (2D 아이콘 → 3D 점진)
```

## PAR/스펙트럼 구현 난이도 (참고, tool_stopline과 연동)
- Lv1 (지금): 광원별 고정 lux→PPFD 계수 + 식물 PPFD 범위 + 스펙트럼 태그 → "여기 PPFD→몬스테라 OK"
- Lv2: R:B 비율 태그 → 잎/개화/무늬 선호 판정
- Lv3: 파장 4~5밴드 모델 (덕후 모드)
- Lv4: 실제 스펙트럼 곡선 + McCree 적분 (정밀 = 종료선 밖, 전문가)
→ **MVP = Lv1 + 약간 Lv2. Lv3~4는 깊게 덕후/나중.**
