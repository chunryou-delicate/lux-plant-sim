## A. 게임 첫 루프 — 뭘 만들 것인가

> 목표: "식물 사고 → 시간 지나면 → 빛 받은 만큼 성장/시듦"을
>       최소 형태로 만들어 "재밌나?" 확인.

### 최소 루프 (MVP)
```
1. 식물 사기   → 도구의 S.plants[] 재사용 (자리 클릭)
2. 시간 흐름   → "하루 넘기기" 버튼 (해석으로 압축, 실시간 X)
3. 빛 판정      → 그 자리 PPFD vs needPPFD (도구 로직 그대로)
4. 성장/시듦   → 적합=성장 / 부족·과광=정체·시듦 (모델 스케일 or 단계 교체)
5. 피드백      → "몬스테라가 무럭무럭! / 빛이 부족해 시들어요"
```

### 첫 프로토 캐릭터 = 자취자 (경제 긴장 되나 · 재미 검증 빠름)
```
- 시작자금 소액 + 월세 압박 (game_design_core.md 참고)
- 식물 키워 팔아야 생존 → "재밌나/짜릿한가" 빠르게 확인
- 단, 첫 루프는 "사고 자라는 재미"부터. 경제는 그 다음.
```

## 착수 순서 (하나씩)
```
STEP A1. 시간 개념 → "하루 넘기기" 버튼 + 날짜 카운터
         · 하루 넘기면 각 식물이 그날 받은 빛(PPFD) 누적
STEP A2. 성장 상태 → 식물마다 성장도(0~100) 필드 추가
         · 적합광이면 +, 부족/과광이면 정체 or -
         · S.plants[i].growth, .health
STEP A3. 시각 피드백 → 성장도 따라 식물 마커 크기/단계 변화
         · 3D 모델 스케일 or 단계별 교체 (씨앗→새싹→성체)
STEP A4. 말 피드백 → "무럭무럭 / 시들해요 / 빛이 너무 세요"
STEP A5. (재미 확인 후) 경제 치기 → 판기·월세, 자취자 긴장
```
→ A1~A4가 "사고 자라는 재미"의 최소 확인.
  재밌으면 → 경제(A5)·캐릭터·이사로 확장.
  안 재밌으면 → 여기서 멈추고 재미 요소 재설계 (엔지니어 함정 회피).
