## 【1단계·도구】 완성 로드맵 (재배열)

### 1-A. 기초 상호작용 최적화 — 지금 여기
> 계산이 정확해도 입력이 불편하면 식집사는 못 씀. 이것부터.

```
① 방 그리기 (막막함 해결) ← 현재 작업
   - 벽 실시간 미리보기 (한붓그리기, 반투명 벽 서기, 뒤로가기)
   - 데이터 구조를 가구부·다중방 대비로 열어둠 (지금은 방 1개만 구현)
   - (선택) 프리셋 방 [네모/원룸/L자]

② 가구 배치 최적화 → 다음
   - 가구 드래그 이동 (지금 없음! 조명만 드래그됨)
   - 가구 회전 / 방 밖 위치 제한·스냅

③ 조명 배치 다듬기
   - 방 밖 이탈 방지, 정렬·스냅 (이미 드래그·종류·높이 됨)
```

### 1-B. 방 구조 확장 (구조 이미 열어둠)
```
④ 벽 뚫기 (가구부) — 문/통로/창을 벽에 뚫기
⑤ 다중 방·통로 — L자·복도, 방 사이 빛 연결
```

### 1-C. 계산·판정 강화 (타겟 직관 우선)
```
⑥ 계산면 여러 개·임의 높이 → 후크 기능 기반
   - 지금: 작업면 1개(바닥 0.75m)
   - → 선반·테이블 높이별 조도 ("높은 선반 밝고 바닥 어둠")
   - 다단 배치 + 게임 "선반 위 식물"로 직결

⑦ 시간대·방위 채광
   - 지금: 천공조도 고정 → 해 위치(시간·계절)·방위 반영
   - "아침 밝고 오후 어둡다" = 식집사가 체감. 기상 데이터 연계

⑧ 조도 → 식물 적합성 → 진짜 후크
   - lx 숫자 대신 "여기 몬스테라 OK / 다육 빛부족" 말로
   - 도구가 게임으로 넘어가는 다리
```

### 1-D. 전문가용 (트랙A, 우선순위 뒤) — 빼는 게 아니라 나중
```
⑨ 등조도 곡선 (isolines)      → 전문가 신뢰도용
⑩ 리포트 출력 (PDF)           → 시공사 산출물용
⑪ 자동 조명 배치 (개선 역산)  → 실무 편의
⑫ 에너지 지표 (W/㎡)          → 실무용
   ※ 이것들은 트랙A(실무 껍데기)에서. 게임/일반 껍데기엔 과함.
```

### 전략적 제외 (절감)
```
✂ 실제 조명 DB(IES)   → 프리셋 근사
✂ CAD/BIM(DWG/IFC)    → 일반 타겟은 도면 없음
✂ 도로·실외·빌보드 등  → 범위 아님
```

## 흔들리지 말 것

- 타겟은 식집사. "쉬움"이 최우선 기능임.

- 구조는 미리 열어둠(가구부·다중방·계산면), 기본 화면은 단순 유지.
- 도구 완성 = "DIALux만큼"이 아니라 "식집사가 우리집 빛+식물 알게".
