# 모듈형 식물 성장 시스템 — 프로시저럴 생성 (2026-07-13)

> ★ 핵심 전략: 모듈 몇 개 + 조합 알고리즘 → 무한 변형.
> 에셋은 최소, 결과는 무한. (마인크래프트가 블록 몇 개로 무한 세계 만들듯)
> "100가지 몬스테라를 만든다" → ✗  "잎·줄기 모듈 + 성장 규칙" ⭐
> ⚠ 강력하지만 구현 난이도 높음 → 코어 기능으로 요약, 단순 버전 먼저.

---

## 1. 왜 모듈화·알고리즘화인가
```
피하는 방식: 몬스테라 100가지 모양 다 제작 → 에셋 100개 (지옥)
하는 방식:   잎·줄기 모듈 몇 개 + 조합 알고리즘 → 무한 변형 (천국)
→ 툴 작고 에셋 부담 큰 우리한테 최적 전략
→ 처음 모듈 세팅만 해두면, 변형은 알고리즘이 알아서 생성
```

---

## 2. 몬스테라 실제 생장 (게임에 반영)
```
- 노드(마디) 생장: 한 마디에서 잎 하나씩. 새 잎이 이전 잎 겨드랑이 근처에서 남
  · ★ 맨 위에서만 순차로 나는 게 아니라, 어느 마디에서든 새 가지가 날 수 있음
- 매번 랜덤 위치·각도 → 개체마다 모양 다름 (똑같은 몬스테라 없음)
- 어릴 땐 통잎 → 자라며 갈라짐(페네스트레이션) = 성숙 신호
- 빛 방향으로 잎이 뻗음 (굴광성)
- 오래된 줄기일수록 굵어짐 (잎이 많이 달릴수록)
```

---

## 3. 모듈 vs 알고리즘 (뭘 만들고 뭘 자동)
### 모듈 = 만들어둔 것 (최소 에셋)
```
- 잎: 3~4종류 (통잎 / 살짝갈라짐 / 많이갈라짐, 크기별)
- 줄기 마디: 1~2종류 (이어붙이는 단위)
- 화분: 몇 종류 (고정, 성장과 무관)
```
### 알고리즘 = 자동 변화 (무한 결과) ★가 에셋이 게임과 연결
```
요소      랜덤/규칙                        게임 시스템 연결
─────────────────────────────────────────────────────
위치      노드마다 랜덤 +빛방향 가중치      → 빛 (굴광성)
각도      잎마다 랜덤                       자연스러운 흐트러짐
크기      생장환경 반영                     → 빛·물·양분 (돌봄 결과)
색깔      건강도·빛 반영                    → 건강 (시들면 누런 것)
개수      성장도 따라 증가                  → 시간·성장
갈라짐    성숙도 따라 (어릴땐 통잎)         → 성숙 (진짜 몬스테라)
─────────────────────────────────────────────────────
→ 식물 모양만 봐도 "어떻게 키웠는지" 보임 = 애착·개체성
```

---

## 4. 이게 게임에 이로운 이유
```
- 내 식물이 세상에 하나뿐 (랜덤 생장) → "우리 애" 애착
- 다음에 접속할 땐 어떻게 자랐을지 → 키우는 오늘 (하루 넘기기가 기대됨)
- 잘 키운 애 = 크고 진하고 창가로 우아하게 / 방치 = 작고 누렇고 시들
- 성장 개체 = 수집·성장·판매 (류류리조트의 수집 재미)
- 빛이 성장 모양까지 지배 → 훅(빛이 진짜)이 성장과 하나됨
→ 아까 "반발함"의 근본 해결책
```

## 5. 다른 식물로 확장 (모듈 시스템의 위력)
```
몬스테라   = 갈라진 잎 모듈 + 노드 생장
포토스     = 하트 잎 모듈 + 넝쿨 생장
산세베리아 = 뾰족 잎 모듈 + 직립 생장
다육       = 통통 잎 모듈 + 로제트 생장
→ 잎 모듈 + 생장 규칙만 교체하면 종별 대응
→ 79종을 다 만드는 게 아니라, 모듈 세트 + 규칙으로 커버
```

## 6. 구현 순서 (★몬스테라부터 생성)
```
1단계 (먼저·단순): 단계별 통째 모델 — 씨앗→새싹→중간→성체 통째 교체
2단계 (재밌으면·코어): 모듈 시스템 — 잎·줄기 모듈화 + 랜덤 노드 생장
   · 위치=랜덤+빛, 크기=돌봄, 색=건강, 방향=빛, 갈라짐=성숙
3단계: 다른 종 확장 — 잎 모듈·생장 규칙만 교체
→ 1단계 재미 확인 없이 2단계 먼저 만들지 말 것 (엔지니어 함정).

※ 현재 프로토(plant_grow.html)는 임시 도형으로 2단계 로직을 먼저 검증 중.
```

## 7. 에셋 제작에 주는 영향
```
기존 계획: "완성된 몬스테라" 원화 1장 → 3D
모듈 반영: 부품별로 뽑아야 함 — 화분 / 잎 모듈(통잎·살짝·많이갈라짐) / 줄기 마디
→ 단, 1단계(통째)는 완성형으로 뽑아도 됨. 2단계 갈 때 부품별로 다시.
⚠ AI가 "잎 하나만" 잘 이해못할 수 있음 → 참조이미지로 스타일 고정, 3D 후 부품분리 확인
```
