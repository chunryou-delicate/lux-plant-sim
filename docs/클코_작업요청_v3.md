# 클코 작업 요청 — 3D 렌더 착수 (v3)

> 방향 확정: Three.js 저폴리 3D. 프로토타입보다 한 단계 진화한 형태로.
> 이 문서 = 클로드 코드에 줄 작업 지시서.

---

## 0. 지금 상황 (클코가 알아야 할 맥락)

```
확정된 것
├─ 렌더 방식: 2D 캔버스 → Three.js 저폴리 3D (전환 확정)
├─ 조도 엔진: 방좌표계 격자에 lx 계산. 렌더와 무관하게 살아남음.
├─ 무게중심: 게임(식집사)이 메인 / 조도툴은 엔진·출발점
└─ 비주얼: 저폴리지만 조명·재질·색으로 퀄리티 (폴리곤수 ❌)

핵심 원리
  lighting.js (조도엔진)  = 2D든 3D든 그대로 재사용 ★
  렌더 부분만            = 캔버스 → Three.js 로 교체
  → 엔진 안 건드리고 껍데기만 바꾼다
```

**기준 파일**: `room_3d_enclosed.html` (밀폐형 방 + 창 개구부 + 면 숨김, 최신).
이걸 렌더의 출발점으로 삼되, 아래 요구대로 "한 단계 진화"시킨다.
※ 이전 `room_3d_lowpoly.html`(오픈박스)은 폐기. enclosed가 최신 기준.
※ (현 저장소에 두 파일 모두 없음 → 스펙대로 새로 구현, 레퍼런스 이미지 퀄리티 목표)

### ⭐ 3D 방의 필수 스펙 (물리적으로 진짜인 빛)
```
1. 방은 6면 다 존재 (천장·바닥·앞·뒤·좌·우) — 겉엔 안 보여도 다 있음
2. 빛은 오직 창(개구부)으로만 들어옴
   - 뒷벽 = 통짜 ❌ / 창 구멍 뺀 4조각(위·아래·좌·우) ⭕ = 진짜 구멍
   - 벽이 빛을 막아야 창가 밝고 안쪽 그늘짐 (진짜 방 명암)
3. 카메라 각도로 가리는 면 숨김 = ★ 심즈2 스타일 벽 cutaway ★
   - 레퍼런스: The Sims 2 의 벽 내림(cutaway) 방식
   - 위에서 보면 천장 / 아래서 보면 바닥 / 앞 가리는 벽 사라짐
   - ⚠ 핵심: 숨겨도 "빛(그림자)은 계속 막아야" 함
     · Three.js visible=false = 그림자도 꺼짐 → ❌ (빛 새어들어옴)
     · 올바른 방법: 재질 투명화(opacity 0) or castShadow 유지하는 별도 방식
     · 프로토타입에서 opacity 방식 시도했으나 불완전 → 클코에서 제대로 구현
   - ※ 프로토타입(room_3d_enclosed)은 "필요 스펙 증명"용. cutaway는 새로 견고하게.
4. 조명: 태양(창밖 DirectionalLight) + 창 SpotLight + 천장등(옵션 PointLight)
   - 실내 환경광은 약하게 (창 없으면 어두운 게 정상)
   - 천장등 3모드: 자동(야간)/상시/끄기. 끄면 완전히 어둠(잔광 ❌)
   - ⚠ MeshBasicMaterial 주의: 빛 영향 안 받아 항상 빛남 (모니터·화면 등 조심)
```

## 1. 이번 목표: 프로토타입 → v3 (한 단계 진화)

| 항목 | 프로토타입 (v1/v2/3d) | v3 목표 |
|------|----------------------|---------|
| 렌더 | 캔버스 or 기본도형 3D | **Three.js 저폴리 3D + 재질·조명 강화** |
| 조도-렌더 연결 | 각자 따로 | **엔진 격자 lx → 3D에 반영 (통합)** |
| 구조 | 단일 HTML | **모듈 분리 (engine/render/data)** |
| 가구 | 박스·구 조합 | **저폴리지만 형태감 있는 모델** |
| 재질 | 단색 | **재질감(나무·천) + 부드러운 그림자** |
| 데이터 | 코드에 박힘 | **data/ 로 분리 (카탈로그)** |

## 2. 작업 순서

### STEP 1 — 데이터 분리 (가볍게 먼저)
```
목표: ROOM/CATALOG 를 코드에서 data/ 로 빼기 (2D·3D 공통이라 안 버려짐)
- data/catalog.json  : 가구·조명·식물 스펙 (needLux, flux, 크기, 높이…)
- data/room_presets.json : 방 프리셋 (원룸/투룸 + 창 위치)
- 엔진·렌더가 이 json 읽도록
```

### STEP 2 — 조도 엔진 정리 (이미 됨, 확인만)
```
목표: lighting.js 가 렌더와 독립인지 확인
- 입력: 방 크기, 창, 조명 배치, 가구 → 출력: 방좌표 격자 lx[][]
- 렌더가 뭐든(2D/3D) 이 격자만 읽으면 됨
```

### STEP 3 — Three.js 3D 렌더 (본론) ★
```
목표: 3D 렌더 모듈화 + 진화
- src/render3d/scene.js   : 씬·카메라·조명(태양+창스팟+천장등)
- src/render3d/room.js    : 방 6면(밀폐) + 창 개구부(4조각 뒷벽) + 면 숨김 로직
- src/render3d/furniture.js: 가구 저폴리 모델 (형태감 있게)
- src/render3d/lighting_viz.js : 엔진 격자 lx → 바닥 히트맵 텍스처 or 밝기 반영
- 실시간: 해 슬라이더·회전 → 즉시 그림자·빛·면숨김 갱신
- PCFSoftShadowMap (부드러운 그림자) 필수
- ★ 빛은 창으로만 (벽이 막음) / 카메라 각도로 가리는 면 자동 숨김
```

### STEP 4 — 조도 엔진 ↔ 3D 연결 (핵심 통합)
```
목표: "빛이 진짜"를 3D에서 증명
- 엔진이 계산한 격자 lx 를 3D 바닥에 히트맵으로 (토글)
- 동시에 Three.js 실제 조명으로 시각적 빛 (이중 표현)
- 식물 위치의 lx ≥ needLux → 반응 (glow 등)
- ⚠ Three.js 조명은 "보기용", 엔진 격자가 "진짜 판정용" — 둘 구분
```

## 3. 폴더 구조 (v3)

```
byeot/
├─ index.html
├─ data/
│  ├─ catalog.json           가구·조명·식물 스펙
│  └─ room_presets.json      방 프리셋
├─ src/
│  ├─ engine/
│  │  ├─ lighting.js         조도 계산 (E=I·cosθ/d², 그림자, radiosity)
│  │  ├─ daylight.js         창 채광·시간대 태양
│  │  └─ grid.js             방좌표 조도격자
│  ├─ render3d/
│  │  ├─ scene.js            씬·카메라·조명 셋업
│  │  ├─ room.js             방·벽·창·바닥
│  │  ├─ furniture.js        가구 저폴리 모델
│  │  └─ lighting_viz.js     엔진 lx → 3D 시각화
│  ├─ ui/
│  │  ├─ controls.js         슬라이더·회전·시간대
│  │  └─ panel.js            상세설정(진짜 빛·조도수치)
│  └─ main.js                조립
├─ prototypes/               (참고용, 폐기 아님 — 검증기록)
└─ docs/
   ├─ byeot_plan.md          기획 트리 (헌법)
   └─ 클코_작업요청_v3.md      이 문서
```

## 4. 이번에 지킬 원칙 (기획 헌법에서)

- **저폴리로 가되 조명·재질·색에 투자** (폴리곤 늘리기 ❌).
- **엔진과 렌더 분리 유지** — 렌더 바꿔도 엔진 안 건드림.
- **"빛이 진짜"를 3D에서 증명** — 엔진 격자 lx가 진짜 판정, Three.js 조명은 보기용.
- **가구 에셋 = 게임 공용** — 조도툴 가구가 곧 게임 가구.
- 한 번에 하나씩. STEP 순서대로. 각 STEP 후 브라우저 확인.
